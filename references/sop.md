# 变量构造五步 SOP:数据分析的"想什么"

句型表解决"怎么写",这份文档解决"想什么"。Day 2 开场正式讲授一遍(10 分钟),
之后每张决策卡的"理由"环节按这五步追问,直到学员形成本能。
面试官在 Day 4 也按这五类出题——因为真实面试官问的就是这五类问题。

## 第一步:从问题倒推,不从字段顺推

错误顺序:拿到表 → 看有什么字段 → 能算什么算什么。
正确顺序:业务问题 → 用什么指标回答 → 指标需要什么变量 → 变量从哪些字段加工。

例:"用户对物流满意吗" → 指标:差评率按配送时长分层 → 变量:配送天数(原始数据
没有) → 加工:收货时间戳减下单时间戳。

追问话术:"你现在算的这个东西,回答的是哪个业务问题?"

## 第二步:先定分析单位

一行代表什么?订单表一行是一笔订单;要分析"用户",必须先聚合到用户级。
90% 的新手错误(重复计数、率的分母错了、merge 后数字翻倍)根源都在这一步。

本数据集的天然陷阱:customer_id 是订单级的,customer_unique_id 才是用户级。
用前者算"用户数"会高估——这个坑要让学员自己掉进去再爬出来,印象最深。

追问话术:"你这个表里,一行代表什么?"——任何时候学员报出一个数字,都可以用这句验货。

## 第三步:四个算子生成候选变量

业务分析的变量构造翻来覆去就四招:

| 算子 | 例子 | 句型 |
|---|---|---|
| 时间差 | 配送天数、复购间隔、下单到发货时长 | 日期相减 + .dt.days |
| 比率 | 差评率、复购率、客单价(注意分母是谁!) | 布尔均值 / 两列相除 |
| 分箱 | 价格带、配送速度档、新老客 | pd.cut / pd.qcut |
| 聚合 | 用户总单数、首末单时间、品类销售额 | groupby + agg |

RFM(最近一次消费 Recency / 频次 Frequency / 金额 Monetary)= 时间差 + 聚合 + 分箱
的经典组合,面试官一听就懂,Day 3 作为标准动作完成。

## 第四步:筛变量三问(不是统计筛选)

不教 p 值、相关系数矩阵、IV 值。业务分析筛变量就三问:

1. **机制故事讲得通吗?** 配送慢→体验差→差评,通;用户 ID 尾号→差评,不通。
2. **有区分度吗?** value_counts 一看 95% 是同一个值 → 扔。
3. **数据质量撑得住吗?** 缺失超三成的字段,用之前必须先解释缺失为什么发生。

可解释性高于一切:每个进入分析的变量,都要能用一句话回答"为什么看它"。

## 第五步:对比留一手,防混杂

任何"A 组比 B 组高"的结论,强制自己按一个关键维度分层再看一遍。

本数据集埋着一个真实案例供 Day 3 使用:整体差评率的品类差异,有一部分
其实是配送时长差异带来的(重货品类配送慢)。学员得出"某品类口碑差"时,
追问:"控制配送速度之后,这个品类还差吗?"——分层后结论会修正。
这一问就是普通候选人和让面试官眼前一亮的候选人之间的分界线。

追问话术:"有没有可能不是 A 导致的,而是 A 和 B 恰好绑在一起?"

## 附:统计问题的标准答案(学员常问)

- **"分组对比要不要做检验?"** 业务判断规则:**效应量优先于显著性**。每组样本上千、
  差距数倍时,检验是仪式不是瓶颈。面试加分句:"严格起见可做卡方检验确认"。
  检验真正必要的场合:小样本、小效应、AB 实验读数。
- **"用方差分析吗?"** 差评率是**比例**,组间比例差异对口**卡方检验**;
  ANOVA 比的是连续变量均值。
- **"为什么报中位数不报均值?"** 配送天数、客单价均右偏,均值被长尾拖高;
  给业务方报中位数,两者差距大时并报并解释。
- **机制证据五件套**(学员问"怎么证明A导致C"):传导链各环直接验证 + 剂量反应梯度 +
  分层稳健 + 排除替代解释 + 承认观察数据边界并提"AB实验可确证"。
